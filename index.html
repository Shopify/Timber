1.1 Who Is This For

Timber is a front-end framework which makes building Shopify themes quick and easy. It can be used by theme creators of any skill level for themes of any scope.

Seasoned pros and newbies alike can benefit from the starter templates, liquid markup, modules, and CSS frameworks provided in Timber.

1.2 What is Included

Required Templates

Every template your store needs is included in one handy place, so no more sticky notes with checklists on your desk.

Basic Markup

Jump past the learning curve of liquid template tags because we've provided you with the essentials.

CSS Framework

Grounded in Sass, Timber provides you with a barebone set of styles that you can quickly put your designer touch on.

2.1 Sass

Timber is built with Sass, but you don't have to know it to use it.

The compiling is all done on the Shopify server, so edit the .scss.liquid file just as you would CSS and watch the magic happen on your site.

Imports and Mixins

You cannot use @imports in a Sass liquid file. That means no dependence on Compass, Bourbon or similar Sass frameworks.

A section labeled Sass Mixins has been included in timber.scss.liquid with a few helpers to start you off.

2.2 Breakpoints

  • Example
  • Code

Define your breakpoints once, and Sass handles the rest.

All breakpoints are defined in the Breakpoint and Grid Variables section of timber.scss.liquid. Easily change, add, or remove the breakpoints and names to suit your needs.

Default breakpoints:

  • Small: 480px
  • Medium: 768px
  • Large: 769px

$breakpoints is broken down into four lines, each a breakpoint: 'small', 'medium', 'medium-down', and 'large'. These are the names used in your classes to define each column width.

Example: A .grid-item div with a class of large--one-half will span 50% of its parent above 769px. Any screen size below that breakpoint will default to (mobile-first) 100%.

$viewportIncrement: 1px;

$small: 480px;
$medium: 768px;
$large: 769px;

$postSmall: $small + $viewportIncrement;
$preMedium: $medium - $viewportIncrement;
$preLarge: $large - $viewportIncrement;

/* The following are dependencies of csswizardry grid */
$breakpoints: (
  'small' '(max-width: #{$small})',
  'medium' '(min-width: #{$postSmall}) and (max-width: #{$medium})',
  'medium-down' '(max-width: #{$medium})',
  'large' '(min-width: #{$large})'
);

Class Names

  • A grid-item will default to a 100% width
  • Class names use an easy to read variation of the BEM naming convention
  • Class name format: breakpoint--column-width - e.g. large--one-half

Silent Classes

If you don't know what silent classes are, you can ignore this section.

The biggest point of contention for many grid systems is how many classes are required in your markup. With Sass, you can use silent classes to handle all .grid-item widths in your stylesheet.

Set $use-silent-classes to true in timber.scss.liquid, and use the following code to set the widths:

.foo {
  @extend %grid-item;
  @extend %small--one-half;
  @extend %large--one-quarter;
}

2.5 Sass Mixins

Shopify liquid files don't support @import. This means you can't rely on helpers like Compass or Bourbon. Instead, there is a section called Sass Mixins in timber.scss.liquid that provides a few of these helpers that you may be looking for.

Breakpoint Mixin

Use this simple mixin to create media queries. Breakpoints are defined under Breakpoint and Grid Variables in timber.scss.liquid. This mixin can wrap class declarations (Example 1 in Code tab), or be placed within them (Example 2 in Code tab).

Mixin source: http://blog.grayghostvisuals.com/sass/sass-media-query-mixin/

Note: Standard media queries will obviously work as well if you'd rather not use the at-query mixin.

@mixin at-query($constraint, $viewport1, $viewport2: null) { ... }
  • Parameters
  • Code
Name Type Description
$constraint String Options: min-width, max-width, or null
$viewport1 String If $constraint is set to max-width:
  • $viewport1 will be your media query's max-width
If $constraint is set to min-width or null:
  • $viewport1 will be your media query's min-width
$viewport2 String If set, this becomes the media query's max-width
Default: null

Example 1

@include at-query('max-width', $small) {
  .foo {
    font-size: 0.8em;
  }
}

Output:
@media screen and (max-width: 768px) {
  .foo { font-size: 0.8em; }
}

Example 2

.foo {
  @include at-query(null, 480px, 768px) {
    font-size: 0.8em;
  }
}

Output:
@media screen and (min-width: 480px) and (max-width: 768px) {
  .foo { font-size: 0.8em; }
}

4.1 Ajaxify Cart

Being teleported to the cash register in a grocery store after picking up a can of soup doesn't make a ton of sense, so why should your customer end up on a /cart page online?

Using the Ajaxify Cart plugin allows a user to add a product to their cart without leaving the current page. The cart will appear as a drawer from the top of the page, a modal, or an inline confirmation.

Note: cart.liquid is still a required template and its markup structures the cart in the drawer/modal.

  • Parameters
  • Code

The cart plugin is enabled by simply calling ajaxifyShopify.init({}) after jQuery is loaded. Pass in the parameters below to customize it.

Plugin Format

ajaxifyShopify.init({ [options] })

Parameters

Name Type Description
options Object A collection of parameters that control the cart plugin, each listed out below.
method String The type of cart you want to enable.
Options: drawer, modal, and flip
Default: drawer.
wrapperClass String The class name that defines your site width, without the leading period.
E.g. wrapperClass: 'site-width'
Default: null.
formSelector Selector Your add to cart form selector from the product page.
Default: form[action^="/cart/add"]
addToCartSelector Selector Your add to cart button selector from the product page.
Default: input[type="submit'] within formSelector
toggleCartButton Selector Toggle the cart open and closed with this selector.
Default: null
cartCountSelector Selector Update the number of items in the cart.
E.g. cartCountSelector: '.cart-count'
Default: null
cartCostSelector Selector Update the total cart cost when products are added or removed.
E.g. cartCostSelector: '.cart-cost'
Default: null
useCartTemplate Boolean Set to true to use the markup from your cart.liquid template. We'll take care of the selector.
Set to false to use the included cart layout, generated by Handlebars.js.
IE8 and below will default to false as they cannot use jQuery's .load event.
Default: false
btnClass Selector If useCartTemplate is set to false, include your standard button class so we can style the Checkout link the same way.
Default: null
moneyFormat String Pass in your shop's currency with the liquid tag {{ shop.money_format | json }}.
Default: '${{amount}}'
disableAjaxCart Boolean Set to true if you want to disable the cart but still use the JS quantity selectors (see below).
Default: false
enableQtySelectors Boolean Replace <input type="number"> elements with a JS quantity selector on your product and cart pages. Set to false to disable this and use the native elements.
Default: true
prependDrawerTo Selector The selector for which element the drawer cart is appended to.
E.g. prependDrawerTo: '#page'
Default: body

Required Files

Include the CSS in your document head and JavaScript file after the inclusion of jQuery in your theme.liquid template.

{{ 'ajaxify.scss.css' | asset_url | stylesheet_tag }}
{{ 'ajaxify.js' | asset_url | script_tag }}

Initialize

<script>
jQuery(function($) {
  ajaxifyShopify.init({
    method: 'flip',
    wrapperClass: 'wrapper',
    formSelector: '#addToCartForm',
    addToCartSelector: '#addToCart',
    toggleCartButton: '#cartToggle',
    useCartTemplate: true,
    btnClass: 'btn'
  });
});
</script>

Horizontal

To make form elements flow inline, add a class of .form-horizontal to the parent.

You cannot add a class directly to most liquid form elements (e.g. {%raw%}{% form 'contact' %}{%endraw%}). Instead, wrap the form in a div with the class .form-horizontal to achieve the same effect.
